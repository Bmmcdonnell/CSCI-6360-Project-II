\section{Introduction}

In this project, we will explore three different datasets using Neural Networks (2, 3, and 4 layers; and CNN). The three datasets we will be using are:

\begin{itemize}
	\item \textbf{Auto MPG:} This dataset contains information about different cars. The goal is to predict the miles per gallon (MPG) given the other characteristics of the car. The dataset can be found at \url{https://archive.ics.uci.edu/dataset}.
	\item \textbf{California Housing Prices:} This dataset contains information about different housing locations. The goal is to predict the median house price of the area given the other characteristics. The dataset can be found at \url{https://www.kaggle.com/datasets/camnugent/california-housing-prices}.
	\item \textbf{Insurance Charges:} This dataset contains information about different primary beneficiaries of health insurance. The goal is to predict the primary beneficiaries insurance charges given their other characteristics. The dataset can be found at \url{https://www.kaggle.com/datasets/mirichoi0218/insurance}.
\end{itemize}

\subsection{Scalation Hyper-parameter Tuning}

\subsubsection{Auto MPG}

For the 2 layer neural network, batch sizes of 1, 4, 8, 16, and 32 were tested. The best results came from the batch size of 1. This should not be surprising, as it allows us to fit the data the closest. Since the Auto MPG dataset is fairly small, a batch size of 1 does not drastically increase the run time of the code. Next, we tested learning rates of 10, 1, 0.01, and 0.001. 0.001 yielded the best results, so we used that as the learning rate.

For the 3 later neural network, multiple different combinations were attempted. The initial parameters tested were: learning rate = 0.01, batch size = 16, activation function on the hidden layer = geLU, activation function on the output layer = reLU, and number of nodes for the hidden layer = $2\times \text{x.dim2}$. We then proceeded to adjust one parameter at a time to see if the results would improve. After several iterations, the best results came from: learning rate = 0.01, batch size = 8, activation function on the hidden layer = sigmoid, activation function on the output layer = reLU, and number of nodes for the hidden layer = $3\times \text{x.dim2}$.

\subsubsection{California House Prices}

For the 2 layer neural network, batch sizes of 32 and 64 were tested. The best results came from the batch size of 32. Next, we tested learning rates of 10, 1, 0.01, and 0.001. 10 yielded the best results, but were still unsatisfactory, so we then tested 20, 25, and 30. 25 yielded the best results, so we used that as the learning rate.

For the 3 later neural network, multiple different combinations were attempted. The initial parameters tested were: learning rate = 0.01, batch size = 32, activation function on the hidden layer = lreLU, activation function on the output layer = id, and number of nodes for the hidden layer = $16$. We then proceeded to adjust one parameter at a time to see if the results would improve. After several iterations, the best results came from: learning rate = 0.01, batch size = 32, activation function on the hidden layer = lreLU, activation function on the output layer = id, and number of nodes for the hidden layer = $3\times \text{x.dim2}$.

\subsubsection{Insurance Charges}

For the 2 layer neural network, batch sizes of 32 and 64 were tested. The best results came from the batch size of 32. Next, we tested learning rates of 10, 1, 0.01, and 0.001. 10 yielded the best results, but were still extremely unsatisfactory, so we then tested 50, 100, 150, and 200. 200 yielded the best results, so we used that as the learning rate.

For the 3 later neural network, multiple different combinations were attempted. The initial parameters tested were: learning rate = 0.01, batch size = 32, activation function on the hidden layer = lreLU, activation function on the output layer = id, and number of nodes for the hidden layer = $\text{x.dim2}+1$. We then proceeded to adjust one parameter at a time to see if the results would improve. After several iterations, the best results came from: learning rate = 0.01, batch size = 16, activation function on the hidden layer = lreLU, activation function on the output layer = id, and number of nodes for the hidden layer = $2\times \text{x.dim2}$.

\subsubsection{Hyper-parameter Tuning}
To optimize performance, we tuned the model by varying the number of filters, activation functions, learning rates, and mini-batch sizes. Specifically, we evaluated different numbers of filters for Layer 1 and Layer 2: (4, 8), (8, 16), (16, 32), (8, 8), and (16, 16). \\
\\
For activation functions, we evaluated ReLU, Tanh, and their combinations in the CNN layers. We also tested various learning rates to identify which produced the lowest validation Mean Squared Error.\\ 
\\
Finally, we evaluated mini-batch sizes of 2, 4, 8, 16, and 32 to determine the optimum number of samples for gradient approximation. These parameters were individually optimized for each dataset. The Adam optimizer was used in all cases.
